## OSS・コミュニティなど

技術コミュニティへの貢献として、これまで複数の技術カンファレンスやミートアップイベントにおいて、主にティザーサイトの設計・実装や、開発チームのリードとして携わってきました。特に 2021 年は、コロナ禍の影響でリモート中心の生活が続いていたこともあり、比較的時間を確保しやすく、これまで以上に技術コミュニティへ深く関わるきっかけとなりました。

中でも、Flutter をテーマにした技術カンファレンス「FlutterKaigi」には、2021 年の立ち上げ当初から関与させていただいており、初回からティザーサイトの設計・実装を担当しました。以降も継続して参加し技術面のリードのみならず、今となってはコアスタッフの中央統括として、プロジェクト全体の時間管理やタスクマネジメントに注力しています。同じ 2021 年には、VS Code Meetup のコミュニティイベント「VS Code Conference JP」で初めてティザーサイト制作の必要性が高まり、その設計・実装を担当しました。Next.js（SSG）と GitHub Pages による構成を採用し、加えて将来的な継続利用を見据えて、モノレポ化や共通コンポーネントの整理にも取り組んでいます。

あと Vue.js のコミュニティイベント「Vue Fes Japan」では、2019 年の台風による中止を外部から目の当たりにし、OSS やコミュニティの運営者として何か力になりたいという思いから、コアスタッフとして参画しました。2022 年より 3 年連続でティザーサイト制作を担当し、Nuxt.js（SSG）と Netlify を用いて構築しています。

このように、それぞれのイベントにおいて特性や要件に応じた技術を選定し、Flutter や Nuxt.js、Next.js といった多様なフレームワークの利活用を進めてきました。フロントエンドの実装だけでなく、チーム運営や再利用性を見据えた構成設計など、裏側の基盤づくりにも注力しています。

## 個人ブログ

知見を発信するため、Nuxt 製のウェブログとして 2018 年 10 月より個人ブログの運営を開始。2023 年春先にかけ Lume 製へリプレースを進めました。

技術カンファレンスをオーガナイズする中で自身は何たる人物か紹介する場面が増えていることが挙げられます。

なお、書き味については Zenn の良さを取り入れるため手元の環境で執筆、公開したいタイミングに GitHub へ push することを目指しました。

#### EventIn リニューアル推進

当初は npm プラグインとしての配布も視野に入れていましたが、プロダクトの性質や開発体制のスケールを踏まえ、現在は「npm パッケージとして切り出す」よりも、プロダクト内部で再利用性の高い汎用化プラグインの開発に重きを置く方針へと切り替えました。特に、モノレポ構成を採用しているプロジェクトにおいては、プラグインや共通モジュールを Nx の libs/shared 配下に集約し、機能単位で厳密に管理・共有しています。これにより、実装の重複や仕様のブレを防ぎつつ、チーム全体での開発効率を高めることができました。

#### [既存案件] イベント配信 EventIn プロジェクト (CRA/Express)

プロダクト開発においては、交流促進と開発体験（DX）の改善を主な目的として、さまざまな施策に取り組みました。なかでも今回の開発では、大規模なリニューアルが至上命題として掲げられ、限られた期間内でいかに高い完成度を実現できるかが大きなチャレンジでした。しかし、十分な開発時間を確保することは難しく、スケジュールやリソースには常に制約がある状況でした。そうした中でも最大限の成果を出すために、開発効率の最適化や工数削減を目的としたさまざまな工夫を取り入れました。

具体的な施策としては、以下のような取り組みを実施しました。

交流施策
　- ユーザー同士のつながりをサポートする「繋がり機能」を新たに実装、利用者間の継続的なエンゲージメントを促進する基盤を整備。

DX（Developer Experience）改善施策
　- Vite の導入による開発環境の刷新：ビルドやホットリロードの高速化を実現し、日々の開発効率を大幅に改善。
　- バリデーションエラーの表示方法の拡充：より明確で扱いやすいエラー表示により、開発者・ユーザー双方の体験を向上。
　- コンポーネント設計の抜本的な見直し：既存構成を根本から再設計し、保守性・再利用性を向上。
　- 型定義に関する開発方針の策定：チーム内での実装統一と型安全性の確保を目指した方針を明文化。

これらの施策に加え、「短期間で高品質なリニューアルをやり遂げる」というプロジェクトの制約下において、CI/CD フローの最適化やチームの生産性向上を図るプロセスも推進しました。

#### [既存案件] セルフオーダー用 Web サイトプロジェクト (Laravel/PHP)

セルフオーダー機能の拡充施策においては、フロントエンドエンジニアとしてだけでなく、インフラやバックエンド領域にも積極的に関与しながら、デザイン設計から実装までを担当しました。

今回の施策では、タブレット端末（特に iPad）での利用を想定した PWA（Progressive Web App）としての UI 設計・最適化に注力しました。注文操作がスムーズに行えるよう、タッチ操作や画面遷移、レスポンシブ対応を意識して設計し、ユーザー体験の向上を図りました。

また、フロントエンドにとどまらず、一 SRE エンジニアとして、AWS Lambda と API Gateway を用いたバックエンド開発にも関与。ユーザー操作に応じてトリガーされる Push 通知連携や Post API の設計・実装を担い、機能としての信頼性とリアルタイム性を両立させる仕組みを構築しました。

これに伴い、通知の疎通確認や処理の安定性を担保するために、インフラチームとの協働によるテスト計画・実施にも参加。開発と運用の間に立つ形で、システム全体の整合性とパフォーマンスに責任を持って取り組みました。

#### [既存案件] 販売時点情報管理 (POS) フロントエンド刷新プロジェクト (CakePHP/PHP)

プロダクト全体の品質向上を目的に、UX 改善、ページ表示速度の最適化、API 仕様の明文化、そして既存フロントエンド基盤の整理といった複数の施策に取り組みました。

まず、UX 改善施策として、フロントエンドのリプレースの一環でデザイン設計から実装までを担当しました。具体的には、ユーザーが複数の店舗を直感的に選択できるようにするモーダル UI の設計・実装や、柔軟な日付範囲の検索を可能にする開始日・終了日の選択コンポーネントなど、ユーザビリティに直結する重要な UI パーツを開発。現場での利用シーンを意識しながら、操作のわかりやすさ・視認性を重視した設計を目指しました。

また、ページ表示速度の改善にも取り組み、ユーザー体験を損なう原因となっていたブロッキングレンダリングの軽減に注力。CSS や JavaScript のサイズ削減、読み込み順序の最適化など、リソースの見直しを通じてパフォーマンス改善を図りました。

さらに、開発チーム全体の共通認識の基盤として、Web API の仕様書（Swagger ドキュメント）を新たに策定。API 名やフィールド名、コメントの整合性を担保するためのテストも自動化し、仕様と実装の乖離を防ぐ仕組みを整えました。これにより、バックエンド・フロントエンド間の連携精度が向上し、開発スピードの安定化にも寄与しました。

加えて、既存のフロントエンド基盤の整理にも着手。特にブラウザごとの表示差異や制約に対応するため、コンポーネントやスタイルを調整し、幅広い環境で安定して動作するように調整を重ねました。
